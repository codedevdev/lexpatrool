**LexPatrol** — десктопный справочник для роли в **государственных службах** в многопользовательском RP (часто на базе GTA V: полиция, шериф, EMS и другие структуры — как принято на вашем сервере). Вы сами наполняете базу: кодексы, уставы и правила с сайтов или своих текстов. Дальше — быстрый поиск по статьям и **оверлей**: отдельное окно поверх игры с закреплёнными материалами.

Приложение **независимое**: не связано с Rockstar Games, издателями GTA и не «официальный» клиент какого-либо RP-сервера — это инструмент для хранения **ваших** импортированных материалов.

**Lex** — про нормы и право; **Patrol** — про патрульную работу.

---

## Что внутри

| Раздел | Зачем |
|--------|--------|
| **База знаний** | Документы и статьи в одном месте, полнотекстовый поиск. |
| **Импорт** | Добавить текст или страницу с сайта в базу — ваш источник правды по серверу. |
| **Читатель** | Удобное чтение статей, закладки, вынос нужного **на оверлей**. |
| **Оверлей** | Окно поверх игры: закрепы, поиск по базе, настройка прозрачности и горячих клавиш. |
| **Заметки** | Свои пометки и привязка к статьям. |
| **ИИ** | По желанию — подсказки по текстам из вашей локальной базы (настраивается в приложении). |
| **Настройки** | Горячие клавиши, оверлей, резервная копия базы, проверка обновлений. |

Есть вариант **portable**: можно держать программу и данные в одной папке и переносить между ПК.

## ИИ-ассистент: настройка и как это работает

ИИ в LexPatrol **не подставляет законы из интернета** и не является «юристом в реальной жизни». Он опирается только на **фрагменты статей из вашей локальной базы** (то, что вы импортировали) и помогает сформулировать ответ для сценария **GTA V RP / FiveM** и похожих серверов. Всегда перепроверяйте формулировки по полному тексту в **читателе**.

### Первый запуск (мастер)

После установки откройте раздел **ИИ**. Мастер предложит:

1. **Провайдер** — куда отправлять запросы (облачный API или локальный сервер).
2. **Агент (по желанию)** — можно выбрать **готовый шаблон роли** (патрульный, инструктор, EMS и др.) или написать свои инструкции; шаблон всегда можно изменить вручную.
3. Сохранение настроек.

Пропустить мастер можно, но без провайдера запросы к ИИ не уйдут.

### Провайдер и модель

| Вариант | Когда удобно | Кратко |
|--------|----------------|--------|
| **OpenAI-compatible** | Локальный **LM Studio** или другой сервер с API в стиле OpenAI | В LexPatrol укажите Base URL вида `http://127.0.0.1:1234/v1`, модель — как в LM Studio. Поле **API-ключ** при локальном сервере часто не проверяется: можно ввести любую **заглушку** (например `lm-studio`), иначе кнопка «Далее» не активируется. |
| **Ollama** | Модель через Ollama на этом или другом ПК | Ключ не нужен; указывается хост и порт Ollama. |
| **OpenAI / Anthropic / Gemini** | Облако, готовые модели провайдера | Нужен **реальный** API-ключ с сайта провайдера; оплата по тарифам сервиса (актуальные цены — только на их сайте). |

**Лимит токенов ответа** (в полной настройке провайдера): если ответ **обрывается на полуслове**, увеличьте значение (для LM Studio нередко 4096–8192).

Подробные пошаговые инструкции (ChatGPT vs LM Studio, лёгкие модели, ссылки) — в самом приложении: **ИИ** → раскрывающийся блок **«Как подключить ИИ и модель»**.

### Агенты

**Агент** — это роль и стиль ответа: дополнительный текст добавляется к общим правилам LexPatrol. В **полной настройке** → вкладка **«ИИ-агенты»** можно добавить несколько агентов и переключать их в «Быстром вопросе».

Готовые шаблоны доступны **в мастере** на шаге агента и при создании **нового агента** во вкладке агентов: нажатие подставляет название, описание и блок инструкций; всё можно отредактировать.

### Ответ и ссылки на статьи

Модель просят ссылаться на статьи в виде `id=<uuid>` из контекста. В интерфейсе LexPatrol такие места показываются **кнопками с названием статьи**; ниже список **«Упомянутые в ответе статьи»** с переходом в **читатель**.

### Где всё это в интерфейсе

- **«Как это работает»** — от импорта до ответа ИИ, вкладки настроек.
- **«Как подключить ИИ и модель»** — ChatGPT/OpenAI, LM Studio, примеры относительно лёгких моделей, ориентир по стоимости.

## Антивирусы и VirusTotal

Если вы прогоняете установщик или exe через [VirusTotal](https://www.virustotal.com/), иногда встречается детект вроде **W32.AIDetectMalware** (например, у **Bkav Pro**). Это не «нашли троян», а срабатывание **эвристики / модели антивируса**: по признакам упаковки, Electron и т.п. файл **похож** на то, что когда-то классифицировали как подозрительное. Сканеры **не читают исходный код** с GitHub и не проверяют, что бинарник собран из открытого репозитория.

Обычно такой детект даёт **один** движок из десятков — для массового вредоноса типична картина с **множеством** срабатываний. У нас **открытый репозиторий**: при желании можно сверить код и процесс сборки.

Если остались вопросы или кажется, что что-то не так — напишите в [**Issues**](https://github.com/codedevdev/lexpatrool/issues).
